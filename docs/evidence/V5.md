# V5 Evidence — Publishing Platform / Lumina Space 1.0

## Delivered journey (verified locally, end to end, 2026-07-16)

`review draft → publish → open stable link anonymously → edit draft → verify live
unchanged → republish → verify link advances → unpublish → 410 → republish → live again`

## ADR 0005 invariant record (e2e run)

| Step | Result |
|---|---|
| Publish | slug `owner-review-exhibition-…`, revision 1 |
| Anonymous public read (no cookies sent) | 200, correct title, 3 photos |
| Edit draft title | public title/revision **unchanged** |
| Draft state | `draftChangedSincePublish: true` |
| Republish | **same slug**, revision 2, edited title now live |
| Delete a published photograph | **409 referenced** (DB-level FK backstop as well) |
| Unpublish | public read **410**; revisions retained |
| Unknown slug | **404** |
| Republish after unpublish | revision 3, public 200 again |

UI verification: editor shows "Live: revision N — up to date / has unpublished
changes", the public link, Publish → Republish → Published ✓ transitions, and
Unpublish. The public page `/e/:slug` renders the revision in the full 3D gallery
(title + description overlay, pointer/keyboard journey verified, gesture opt-in
present, 2D fallback path shared with the main app) with **no account surface at all**.

## Engineering decisions recorded

- **Atomicity without transactions:** neon-http cannot run transactions, so publish
  writes the revision + snapshot rows first and advances the publication pointer as
  the final single UPDATE. A mid-publish failure can only orphan an unreferenced
  revision — the public link never sees a partial state.
- **Database clock everywhere:** `draftChangedSincePublish` compares `updatedAt`
  with `publishedAt`; both now come from Postgres `now()`. The original Worker-clock
  write produced a false negative under clock skew — caught by e2e, fixed the same day.
- **`Cache-Control: no-store`** on the public API: correctness before caching while
  presigned URLs expire hourly and republish must be immediately visible; revisit
  with deployment traffic.
- **Snapshot denormalisation:** revision photo rows copy title/caption/dimensions, so
  a published revision stays exactly what was published regardless of later edits.
- **Routing:** `/e/:slug` is a hand-rolled path match; a router dependency arrives
  with a third route, not before.
