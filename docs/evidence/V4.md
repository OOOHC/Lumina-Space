# V4 Evidence — Template exhibition editor

## Delivered journey

`create → choose library photos → arrange → edit title/text → preview draft in the
real 3D gallery → return and continue editing`, plus archive/restore lifecycle and
publication-readiness diagnostics.

## Exit criteria record

- **Creation + preview without developer assistance:** the full journey was driven
  through the UI in-browser on 2026-07-16: create "Owner Review Exhibition" → add three
  library photographs → reorder (←/→) → set cover (★ follows the photo through
  reorders) → Preview in gallery. The preview hung the owner's own R2-hosted
  photographs on the 3D walls; the detail overlay credited "Draft preview — Owner
  Review Exhibition"; Exit preview restored the sample exhibition. The V1 intent seam
  meant zero changes to scene/gallery code for any of this.
- **Autosave:** 700 ms debounced writes; the Saving/Saved/Failed line reflects the
  server echo (never optimistic), Failed offers retry. Verified live including the
  failure path (see problem record below). Readiness (missing-title / no-photos /
  archived) is a pure, unit-tested function surfaced verbatim in the UI.
- **Preview = last saved server draft** by construction: preview consumes the photo
  list from the most recent save response, not local state.
- **No arbitrary scene edits:** the editor exposes order, cover, title, description —
  nothing else exists.

## Recorded simplification

Leave-page protection is the debounce + visible save state; there is no beforeunload
interception. A keystroke burst followed by an instant tab-close inside 700 ms can lose
that burst. Parked with the beforeunload idea in PARKING_LOT.

## Problems found and fixed during V4

1. **Autosave silently dead on first UI run** — CORS `allowMethods` lacked PUT (DELETE
   had been added earlier; the new verb was forgotten). Preflight returned 204 but the
   browser refused the PUT with `net::ERR_FAILED`. Fix: PUT added; lesson recorded —
   a route's HTTP verb is part of its CORS surface.
2. **3D textures cannot use the credentialed `/view` redirect** — texture loaders fetch
   with `crossOrigin=anonymous` (no cookies). Exhibition responses now embed short-lived
   presigned preview/thumb URLs per photo. Rejected alternative: authenticated XHR +
   blob URLs (more moving parts, worse caching, no security gain).
