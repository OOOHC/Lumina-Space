# V1 Evidence — Stable local viewing gallery

Recorded before gate approval, as required by the V1 exit criteria.

## Performance budget (binding for the V1 gate)

| Metric | Budget | Measured 2026-07-15 |
|---|---|---|
| JS bundle (gzip) | ≤ 500 KB | 303 KB |
| Stylesheet (gzip) | ≤ 10 KB | 1.2 KB |
| Total photo payload (12 images) | ≤ 4 MB | ~0.62 MB (42–63 KB each) |
| Frame rate on mid-tier desktop | 60 fps target, ≥ 30 fps floor | smooth orbit observed in Chromium; formal fps capture pending user hardware run |
| JS heap after 15 open/close cycles | no unbounded growth | 49 MB, stable |

Known deviation: the un-gzipped JS chunk is 1.1 MB (three.js), which triggers the
bundler's 500 KB chunk warning. Accepted for V1; code-splitting is not justified until
there is a second route.

## Manual journey record — 2026-07-15

Environment: Chromium (Claude browser pane), Windows 11, viewport 1280×720, DPR 1.5,
WebGL2 available, `npm run dev` local server.

3D journey (`enter -> orient -> browse -> open detail -> read metadata -> return -> reset`):

- Enter: loading screen resolved to the gallery room; 4 photographs visible on the
  facing wall, keyboard focus ring on the first.
- Orient: pointer drag orbited the camera; walls stayed enclosing; zoom-out clamped at
  max distance (camera cannot leave the room or clip walls).
- Browse: ArrowRight moved the focus ring photo to photo with a polite live-region
  announcement for screen readers.
- Open detail: both pointer click and Enter opened the detail overlay ("Dawn Meridian",
  "Ember Horizon" verified) with dominant image, caption, location/year, credit.
- Return: Escape and the "← Back to gallery" button both closed the overlay.
- Reset: R and the "Reset view" button returned the camera to the home view.
- Stability: 15 scripted open/close cycles — 0 failures, heap stable at 49 MB.
- Failure paths: per-photo texture errors fall back to a matte placeholder (frame-level
  error boundary); repository failure shows a designed error screen with a working
  "Try again" (no page refresh required); empty dataset shows a designed empty state.

Fallback journey (viewport 375×812 and 291-px narrow pane):

- Editorial single-column card list rendered with wordmark and collection subtitle.
- Card tap opened the same detail overlay; "Back to gallery" returned. Metadata intact.
- With WebGL absent the same layout is served with a one-line explanatory note.

Console: zero errors or warnings attributable to application code (one deprecation
notice from a three.js internal — `THREE.Clock` — emitted by the R3F library, not by
this codebase).
